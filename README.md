# Task Manager (MVP Refactoring & Extension)
## ⚙️ Требования

**Перед запуском убедитесь, что установлены:**
- Docker
- docker compose
## 🐳 Запуск проекта
### 1. Клонирование репозитория

```bash
git clone <YOUR_REPOSITORY_URL>
cd <YOUR_PROJECT_FOLDER>
```
### 2. Запуск через Docker Compose
``` bash
docker compose up --build
```
### 3. Доступ к приложению
- Frontend: http://localhost:3000
- Backend: http://localhost:8000
- Swagger: http://localhost:8000/docs
## 📌 Описание проекта

Это демо-приложение **Task Manager**, реализованное на стеке:

- **Backend:** Python (FastAPI)
- **Frontend:** React (Vite)
- **Database:** MySQL
- **Infrastructure:** Docker / docker compose

Изначально проект представлял собой MVP с упрощённой архитектурой. В рамках доработки был выполнен:

- Рефакторинг backend и frontend архитектуры
- Улучшение структуры проекта и читаемости кода
- Добавление системы статусов и приоритетов задач
- Реализация системы мониторинга изменений задач
- Добавление страницы статистики и аналитики пользователей
- Оптимизация работы с базой данных
- Улучшение UX/UI интерфейса

---

## 🚀 Функциональность

### 📋 Управление задачами
- Создание / редактирование / удаление задач
- Статусы: `Backlog`, `In Progress`, `Done`
- Приоритеты: `Low`, `Medium`, `High`

### 🔍 Фильтрация
- Фильтрация задач по статусу
- Фильтрация задач по приоритету

### 📊 Аналитика
- История изменений задач (кто, когда и что изменил)
- Отслеживание переходов между статусами
- Расчёт среднего времени нахождения задач в каждом статусе
- Топ-3 самых продуктивных пользователей (по скорости перевода задач в Done)

## 🪄 База данных
#### **При запуске автоматически:**
- создаётся MySQL база
- применяются миграции Alembic
- загружаются тестовые данные (init_mock_data.sql)
